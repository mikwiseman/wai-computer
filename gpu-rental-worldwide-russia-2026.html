API alternative, not GPU rental

Groq Whisper large-v3-turbo is $0.04 per transcribed audio-hour. It is not a T-one/GigaAM replacement for Russian quality, but it is so cheap that it should stay in the comparison for English, fallback Whisper, and burst workloads. A rented always-on $197/mo GPU must process roughly 4,925 audio-hours/month before it beats Groq on raw Whisper-turbo transcription price.

For Russian, the reason to self-host is model quality and control, not just compute cost. We need a direct benchmark on our real recordings before moving production traffic.

Recommended next move

  • Start outside Russia on Hyperstack A4000 for the first benchmark: deploy T-one and GigaAM, run 50-100 real WaiComputer samples, record WER, latency, GPU utilization, cold-start time, and cost per audio-hour.
  • If the benchmark is good and the service must be always-on globally, move to RunPod A5000 24 GB for a cleaner production baseline.
  • If Russia hosting matters, test FirstVDS vGPU for cheap feasibility, then choose CloudCompute RTX 3090 or HOSTKEY A5000/4090 for production depending on persistence, network, and support.
  • Do not buy A100/H100 time until VibeVoice-ASR or training is explicitly in scope.

Acceptance checklist for any rented GPU

  • nvidia-smi shows the expected physical GPU, VRAM, driver, and CUDA runtime.
  • T-one streaming test runs for 30 minutes without dropped chunks or latency spikes.
  • GigaAM batch test processes long recordings without VRAM growth or worker crash.
  • Persistent volume survives stop/start; stopped storage billing is understood.
  • Provider supports private image or reproducible Docker deployment.
  • No silent fallback to CPU. If CUDA is unavailable, fail the health check.
  • Network path to wai.computer backend is stable; PII logging stays off.
